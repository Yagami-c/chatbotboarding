### 八、小瑞 AI 助手（完整流程）

#### 8.1 Onboarding（已优化）
- 首次进入小瑞时，先展示欢迎 + 称呼输入 + 智能模式选择（与全局 Onboarding 联动）。
- 若全局 Onboarding 已完成，则直接进入 Day1 评估。

#### 8.2 5 步评估 Pipeline（与之前一致）

所有评估流程均遵循以下 5 步 Pipeline，仅在内容上有所差异。

```
步骤条（垂直任务拆解）：
✅ 小瑞了解你 → 基本信息 & 评估
⏳ 推荐方案 → 生成初始强度
⏸ 陪伴治疗 → 设备控制（遥控器独立）
⏸ 记录变化 → 使用反馈
⏸ 持续优化 → 调整下次强度
```

#### 8.3 智能模式开启后的额外功能

| 功能 | 说明 |
|------|------|
| **每日主动提醒** | 每天定时推送训练提醒（可设置时间） |
| **自动推送评估** | Day1/Day7 自动引导评估，Day2-6 自动询问日常感受 |
| **科普推送** | 使用完 PAD 后推送相关科普内容（满意度评价） |
| **运动建议推送** | 根据评估结果推送个性化运动视频（满意度评价） |
| **异常预警** | 连续 3 天未完成训练或状态下降，主动建议调整方案 |

#### 8.4 智能模式关闭后的功能

| 功能 | 说明 |
|------|------|
| **被动问答** | 用户主动提问，小瑞回答 |
| **手动触发评估** | 用户通过首页「手动评估」入口填写 |
| **无主动推送** | 不推送提醒、科普、运动建议 |
| **数据仍存储** | 所有使用数据仍正常记录，可在「我的」中查看 |


### 九、悬浮球遥控器（独立硬件控制）

#### 9.1 设计原则
- **独立于所有页面**：悬浮球常驻于所有页面之上（除全屏弹窗外），全局同步设备状态。
- **硬件质感**：轻微透明度 + 磨砂玻璃效果。
- **状态驱动**：悬浮球本身即反映设备连接与运行状态。

#### 9.2 视觉规范

| 属性 | 值 |
|------|-----|
| 尺寸 | 56×56px，正圆形 |
| 位置 | 右侧贴边，垂直可拖拽滑动 |
| 背景 | `rgba(46, 204, 113, 0.2)` + `backdrop-filter: blur(8px)` |
| 边框 | `1px solid rgba(255,255,255,0.3)` |
| 图标 | 28×28px，白色线性图标，线宽 1.5px |
| 指示灯 | 12×12px，右上角，边框 2px white |

#### 9.3 状态表

| 状态 | 背景色 | 图标 | 指示灯 | 动画 |
|------|--------|------|--------|------|
| 未连接 | `rgba(200,200,200,0.2)` | 🔌 | 灰色 | 无 |
| 已连接待机 | `rgba(46,204,113,0.25)` | ✅ | 绿色（静态） | 无 |
| 运行中 | `rgba(46,204,113,0.35)` | ⚙️（旋转） | 绿色（呼吸） | 旋转 2s linear |
| 暂停 | `rgba(243,156,18,0.25)` | ⏸ | 橙色 | 停止旋转 |
